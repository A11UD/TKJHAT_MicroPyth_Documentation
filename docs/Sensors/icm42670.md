# ICM-42670

## Detailed Description
This module provides a MicroPython API for the TDK InvenSense ICM-42670 6-axis inertial measurement unit (IMU) used on the TKJHAT extension.
The ICM-42670 is connected over the I2C bus and provides:

- 3-axis accelerometer
- 3-axis gyroscope
- internal temperature sensor

The driver is intentionally kept simple for teaching purposes: it configures the most common settings and reads all sensor values using one burst read.

**I2C address:** 0x69 (alternative: 0x68)

**Implemented features:**
- Accelerometer + gyroscope configuration (ODR + full-scale range)
- Simple “start with defaults” setup for exercises
- Burst read of temperature + accel + gyro data
- Returns scaled values in user-friendly units (g, dps, °C)
- Minimal, course-friendly API and structure

**Note:** The ICM-42670 supports advanced features like interrupts and FIFO, but these are not implemented in this MicroPython course driver.

---

## Default connections (TKJHAT)

| Signal | Example Pico pin | Description |
|---|---:|---|
| SDA | GPIO12 | I2C data |
| SCL | GPIO13 | I2C clock |
| Address | 0x69 (alt 0x68) | IMU I2C address |

---

## Default configuration used by `start_with_default_values()`

- Accelerometer range: ±4 g
- Gyroscope range: ±250 dps
- Output data rate: 100 Hz (both)
- Power mode: low-noise mode (LN)

---

## Usage example

```python
from machine import I2C, Pin
from icm42670 import ICM42670
import time

i2c = I2C(0, sda=Pin(12), scl=Pin(13), freq=400_000)

imu = ICM42670(i2c)
imu.init()
imu.start_with_default_values()

while True:
    ax, ay, az, gx, gy, gz, t = imu.read_sensor_data()
    print("accel[g]:", ax, ay, az, " gyro[dps]:", gx, gy, gz, " temp[C]:", t)
    time.sleep(0.1)
```

## Notes

### Returned units
- acceleration in **g**
- angular rate in **dps**
- temperature in **°C**

### Conversion formulas
```python
# Accelerometer (g): raw_value / scale_factor
# scale_factor: 16384 (±2g), 8192 (±4g), 4096 (±8g), 2048 (±16g)

# Gyroscope (dps): raw_value / scale_factor
# scale_factor: 131 (±250 dps), 65.5 (±500 dps), 32.8 (±1000 dps), 16.4 (±2000 dps)

# Temperature (°C): (raw_value / 128.0) + 25
```

### Sensor data readout
The driver reads sensor values using a single 14-byte burst read:
- temperature (2 bytes) + accel (6 bytes) + gyro (6 bytes)

## ICM42670 code documentation

### `class ICM42670(i2c, address=0x69)`
Create an ICM42670 sensor instance using an existing MicroPython I2C bus.

**Parameters:**
- `i2c`: initialized `machine.I2C` object
- `address`: I2C address to use initially (default 0x69)

**Note:**
- During `init()`, the driver also checks the alternative address 0x68 if needed.

---

## Function Documentation

### `soft_reset()`
Perform a sensor soft reset.

**What it does:**
- Writes the soft reset bit
- Waits briefly for the device to restart

**Notes:**
- `init()` calls this automatically.

---

### `init() -> int`
Initialize the IMU and verify connection.

**What it does:**
- Soft reset
- Tries to detect the correct I2C address (0x69 or 0x68)
- Verifies `WHO_AM_I` value (must be `0x67`)

**Returns**
- `0` on success
- negative value on error:
  - `-1`: device not found on I2C
  - `-2`: `WHO_AM_I` mismatch
  - `-3`: I2C communication error

**Precondition**
- I2C bus must be initialized
- IMU must be connected and powered

**Postcondition**
- Device is ready for configuration and reading

---

### `startAccel(odr_hz: int, fsr_g: int) -> int`
Configure accelerometer output data rate and range.

**Parameters**
- `odr_hz`: one of `{25, 50, 100, 200, 400, 800, 1600}`
- `fsr_g`: one of `{2, 4, 8, 16}`

**Returns**
- `0` on success
- negative on error:
  - `-1`: unsupported accelerometer range
  - `-2`: unsupported ODR
  - `-3`: I2C error

**Notes**
- Updates the internal accelerometer scaling used by `read_sensor_data()`.

---

### `startGyro(odr_hz: int, fsr_dps: int) -> int`
Configure gyroscope output data rate and range.

**Parameters**
- `odr_hz`: one of `{25, 50, 100, 200, 400, 800, 1600}`
- `fsr_dps`: one of `{250, 500, 1000, 2000}`

**Returns**
- `0` on success
- negative on error:
  - `-1`: unsupported gyro range
  - `-2`: unsupported ODR
  - `-3`: I2C error

**Notes**
- Updates the internal gyro scaling used by `read_sensor_data()`.

---

### `enable_accel_gyro_ln_mode() -> int`
Enable low-noise mode for accelerometer and gyroscope.

**Returns**
- `0` on success
- `-1` on I2C error

---

### `start_with_default_values() -> int`
Configure the sensor using the course defaults.

**What it does:**
- `startAccel(100, 4)`
- `startGyro(100, 250)`
- `enable_accel_gyro_ln_mode()`

**Returns**
- `0` on success
- otherwise returns the first error code encountered

---

### `read_sensor_data() -> tuple | None`
Read temperature + accelerometer + gyroscope measurements.

**Returns**
- tuple `(ax, ay, az, gx, gy, gz, t)` where:
  - `ax, ay, az` are acceleration in g
  - `gx, gy, gz` are angular rate in dps
  - `t` is temperature in °C
- returns `None` if I2C communication fails

**Notes**
- Uses one burst read (efficient).
- At rest, one accelerometer axis should be near ±1 g (gravity).
